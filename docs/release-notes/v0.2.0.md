# v0.2.0 Research Credibility Hardening

发布日期：2026-05-19

## 定位

本版本将项目命名为 **FactorZen**，并强化单因子研究可信度，重点是配置真实生效、数据质量可审计、实验运行可追溯、walk-forward/OOS 结果可见。

它仍不是生产交易系统，不覆盖 tick、实盘 OMS、盘口成交或 CI 中的真实网络依赖 smoke。

## 新增能力

- YAML 配置接入日跑与报告入口，`preprocessing`、`backtest`、`cost_model`、`walk_forward` 字段会影响真实运行。
- 新增数据质量报告 `*_quality.json`，记录样本覆盖、重复键、收益对齐等检查和 warning。
- 新增实验 manifest，记录完整 config、命令、git SHA、dirty 状态、`pixi.lock` hash 和输出路径；失败时也写入 `status=failure` 与 `error`。
- 新增 walk-forward/OOS 摘要，包含 `n_folds`、IS/OOS Sharpe、OOS 最大回撤和稳定性比率；样本不足时记录 `insufficient_data`。
- Tear Sheet 增加 OOS 区块；无 folds 时显示样本不足。

## 质量门

- mypy 覆盖框架包 `src/factorzen`。
- 新增 coverage 任务，初始门槛 70%。
- CI 保留 lint、typecheck、test，并新增 coverage step。
- smoke 保持离线导入检查，不引入真实 Tushare 网络依赖。

## 边界说明

- `src/factorzen/research/combination/` 仍是实验性样本内工具，`ic_weighted` 和 `max_ir` 依赖样本内 IC 估计权重，不能把组合结果称为 OOS。
- 本版本提升研究复现和失败审计能力，但不承诺生产级交易、撮合、风控或执行闭环。
