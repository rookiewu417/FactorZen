# v0.3.0 Platform Upgrade — M0-M7 完整买方研究平台

发布日期：2026-07-01

## 定位

v0.3.0 是自 v0.2.0 以来最大规模的版本升级。项目从「A 股低频单因子研究框架」扩展为**端到端、可复现的 A 股量化研究平台**：因子挖掘 → 防过拟合 → 风险建模 → AI 智能挖掘 → 组合优化与归因 → 模拟交易 → 成果展示，每一步落 manifest、可审计、可复现。

测试套件从 v0.2.0 扩展到 **1109 个 pytest 用例**，全部离线可重复（不依赖真实 Tushare token 或本地 data/）。

---

## M0 · 微观结构与回测约束

**新增能力**

- `core/universe.py` — universe 快照：停牌/涨跌停/ST/次新股/流通市值过滤，日级快照写入 `workspace/universes/`。
- `core/benchmark.py` — 基准管理：支持 HS300、ZZ500、ZZ1000 指数基准，以及行业等权替代基准（`fetch_index_weights` 真实指数权重接口预留，MVP 阶段为等权基准）。
- `daily/evaluation/backtest.py` — 回测引擎强化：
  - GEM 双路径容差（open → vwap 回落≤1 % 时视为可成交），减少因涨停判定过严导致的漏成交。
  - T+1 约束与 overnight/open-to-close 收益对齐。
  - `adv_20d` 容量检测：ADV 缺失或为零时有 fallback 与警告，不静默跳过。
  - `signal_date` 显式字段集成：因子信号日与执行日解耦，防止前视偏差。

**MVP 限制**

- 行业中性基准为相对等权基准，非真实指数成分权重（后者已预留接口）。
- `intraday/` 分钟线路径保留为研究代码，不是当前路线图主线。

## M1 · 因子挖掘引擎

**新增能力**

- `discovery/operators.py` — 算子库（时序/截面/算术/叶子特征共 30+ 算子），支持 group-safe `pct_change`、`ts_rank` 强制 `min_samples`。
- `discovery/expression.py` — 表达式 AST ↔ 字符串双向序列化（parse / to_string / eval 闭环）。
- `discovery/search.py` — 随机搜索 + 遗传进化搜索（`--method random/genetic --trials --top-k --seed`）。
- `discovery/dedup.py` — 贪心去相关：按 IC 降序入选，相关性超阈值的候选直接淘汰。

**CLI**

```bash
fz mine search --start 20200101 --end 20231231 --method genetic --trials 200 --top-k 10
fz mine leaderboard <session_dir>
```

**MVP 限制**

- 遗传搜索目前单进程；大规模并行搜索需外部 tmux 多进程。

## M2 · 防过拟合护栏

**新增能力**

- `validation/bootstrap.py` — block bootstrap IC 置信区间（HAC 相容，支持可配置 block size）。
- `validation/deflated_sharpe.py` — Deflated Sharpe Ratio（DSR）：修正多次独立尝试的 selection bias。
- `validation/pbo.py` — PBO/CSCV（Probability of Backtest Overfitting）：通过组合交叉验证估计回测过拟合概率。
- holdout 段永久隔离：`--start/--end` 之外的数据不参与搜索与验证循环。

**CLI**

```bash
fz validate overfit <factor> --start 20200101 --end 20241231
```

---

## M3 · Barra 风险模型

**新增能力**

- `risk/factor_model.py` — Barra 风格因子暴露（8 个风格因子：Beta、Size、B/M、Momentum、Liquidity、Earnings Yield、Leverage、Growth）+ 行业因子（中信一级）。
- `risk/covariance.py` — Newey-West 协方差估计（支持可配置 `--nw-lags`、`--cov-half-life`）。
- `risk/specific_risk.py` — 特质风险 James-Stein 收缩。
- `risk/mcr.py` — 边际风险贡献（MCR）分解与风险归因。

**CLI**

```bash
fz risk build --start 20200101 --end 20241231 [--cov-half-life 252 --nw-lags 5 --spec-shrinkage 0.1]
```

产物写入 `workspace/risk_models/{run_id}/`。

---

## M4 · 组合优化与归因

**新增能力**

- `portfolio/optimizer.py` — cvxpy 因子形式 mean-variance QP（CLARABEL solver）：
  - box 约束（`--w-max`）、预算约束（权重和 = 1）、换手约束（`--turnover`）。
  - 行业中性 + 风格中性约束（`--industry-neutral`）。
- `attribution/brinson.py` — Brinson 多期归因（配置/选股/相互作用效果分解）。
- `attribution/risk_attribution.py` — 风险因子归因（持仓风格暴露 × 因子收益）。

**CLI**

```bash
fz portfolio build --start 20200101 --end 20241231 --alpha-file alpha.parquet --industry-neutral [--lam 1 --w-max 0.05 --turnover 0.3]
```

产物写入 `workspace/portfolios/{run_id}/`。

**MVP 限制**

- 行业中性约束基于等权基准，非真实指数成分权重。
- 收益归因（Brinson）依赖持仓期收益数据；数据不完整时部分分解项为空。

## M5 · LLM 单 Agent 挖掘

**新增能力**

- `agents/agent.py` — 零外部依赖自建 LLM 闭环（假设 → 生成表达式 → 护栏过滤 → IC 验证 → critic 反思 → 迭代）。
- Negative RAG：把历史失败案例注入 prompt，引导模型跳出重复错误。
- 因子候选落 manifest（表达式 / IC / 迭代号 / 拒绝原因）可审计。

**CLI**

```bash
fz mine agent --start 20200101 --end 20231231 [--iterations 30]
```

## M6 · 多 Agent 团队

**新增能力**

- `agents/roles/` — 5 角色分工：Hypothesis（假设生成）/ Coder（表达式实现）/ Critic（否决审查）/ Librarian（记忆检索）/ Evaluator（IC 评估）。
- `agents/team_orchestrator.py` — 跨轮协调：Critic 可在任意轮否决当前候选并要求重新假设。
- `agents/experiment_index.py` — 跨 session 长期记忆（已探索假设 / 成功因子 / 失败模式 JSON 索引）。

**CLI**

```bash
fz mine team --start 20200101 --end 20231231
```

## M7 · 模拟交易 + 成果展示

**新增能力**

- `sim/runner.py` — 多周期权重回测：读取 `workspace/portfolios/{run_id}` 的目标权重，对齐行情数据，扣除换手成本，输出净值序列与绩效指标（年化收益、夏普、最大回撤、卡尔玛、换手率、信息比率）。
- `sim/metrics.py` — 绩效计算（日/月/年频率转换，相对基准超额收益）。
- `reports/portfolio_report.py` — 组合绩效 HTML Dashboard：指标卡 + 净值曲线 + 月度收益热图 + Brinson 归因条形图 + 风险摘要。

**CLI**

```bash
fz sim run --portfolio-dir workspace/portfolios/<id> --start 20200101 --end 20241231
fz sim show --sim-dir workspace/sim/<id>
fz report portfolio --sim-dir workspace/sim/<id> --portfolio-dir workspace/portfolios/<id> [--out report.html]
```

## 明确边界与诚实说明

1. **行业中性约束**：相对等权基准，非真实指数成分权重。`fetch_index_weights` 接口已预留，接入真实权重可后续扩展。
2. **LLM Agent**（M5/M6）：依赖外部 LLM API，不内置模型；在无 API key 环境下 Agent 命令不可用，其余功能不受影响。
3. **模拟交易**（M7）：基于历史复权价，不考虑盘口深度、部分成交与实盘滑点；不接实盘 OMS，不做实盘下单。
4. **跟踪误差约束**：当前组合优化不内置跟踪误差约束（TEV），需手动通过换手约束近似控制主动风险。
5. **样本内组合合成**：`research/combination/` 仍是实验性工具，样本内 IC 估权重的结果不能称为 OOS 能力。
