# v0.3.0 完整买方研究平台升级

发布日期：2026-07-01

## 定位

v0.3.0 是自 v0.2.0 以来最大规模的版本升级。项目从「A 股低频单因子研究框架」扩展为**端到端、可复现的 A 股量化研究平台**：因子挖掘 → 防过拟合 → 风险建模 → AI 智能挖掘 → 组合优化与归因 → 模拟交易 → 成果展示，每一步落 manifest、可审计、可复现。

测试套件从 v0.2.0 扩展到 **1111 个 pytest 用例**，全部离线可重复（不依赖真实 Tushare token 或本地 data/）。

---

## 微观结构与交易约束

**新增能力**

- `core/universe.py` — universe 快照：停牌/涨跌停/ST/次新股/流通市值过滤，日级快照写入 `workspace/universes/`。
- `daily/evaluation/benchmark.py` — 基准对比：`compute_excess_return` 计算策略相对 HS300、ZZ500、ZZ1000 等真实指数（Tushare 指数日线）的年化超额收益、跟踪误差、信息比率、超额回撤，经 `fz factor run --benchmark`（默认按股票池映射对应指数）接入单因子评估。`core/benchmark.py` 是另一个同名模块，仅用于记录流水线步骤耗时/峰值内存的性能计时，与金融基准无关。
- `daily/evaluation/backtest.py` — 回测引擎强化：
  - 涨跌停判断的浮点比较容差（`1e-9`，按板块阈值：主板 9.8%/创业板及科创板 19.8%/北交所 29.8%），防止开盘涨跌幅出现 19.7999999...≥19.8 这类浮点边界误差导致漏判涨停/跌停。
  - T+1 约束与 overnight/open-to-close 收益对齐。
  - `adv_20d` 容量检测：ADV 缺失或为零时有 fallback 与警告，不静默跳过。
  - `signal_date` 显式字段集成：因子信号日与执行日解耦，防止前视偏差。

**MVP 限制**

- 行业中性基准（组合优化 `--industry-neutral`）为相对等权基准，非真实指数成分权重（当前未实现，无预留接口）。
- `intraday/` 分钟线路径保留为研究代码，不是当前路线图主线。

## 因子挖掘引擎

**新增能力**

- `discovery/operators.py` — 算子库（时序/截面/算术/叶子特征共 30+ 算子），支持 group-safe `pct_change`、`ts_rank` 强制 `min_samples`。
- `discovery/expression.py` — 表达式 AST ↔ 字符串双向序列化（parse / to_string / eval 闭环）。
- `discovery/mining_session.py` + `discovery/search/`（`random_search.py` / `genetic.py`）— 随机搜索 + 遗传进化搜索（`--method random/genetic --trials --top-k --seed`），落 session 产物。
- `discovery/scoring.py` — IC 打分 + 贪心去相关：按 IC 降序入选，相关性超阈值的候选直接淘汰。
- `discovery/export.py` — 候选导出：将单个候选表达式算成单截面 alpha（`fz mine export-alpha`），衔接下游组合优化。

**CLI**

```bash
fz mine search --start 20200101 --end 20231231 --method genetic --trials 200 --top-k 10
fz mine leaderboard <session_dir>
# 把某个候选导出为 (ts_code, alpha) 单截面 parquet，供组合优化使用
fz mine export-alpha --session <session_dir> --rank 1 --date 20231229 --out alpha.parquet
```

搜索产物落 `workspace/mining_sessions/session_{seed}_{method}/`，含 `candidates.csv`（列 `rank,n_trials,expression,ic_train,...`）、`manifest.json` 与 `exported/*.py`。`--trials` 默认 200。

**MVP 限制**

- 遗传搜索目前单进程；大规模并行搜索需外部 tmux 多进程。

## 防过拟合护栏

**新增能力**

- `validation/bootstrap.py` — block bootstrap IC 置信区间（HAC 相容，支持可配置 block size）。
- `validation/deflated_sharpe.py` — Deflated Sharpe Ratio（DSR）：修正多次独立尝试的 selection bias。
- `validation/pbo.py` — PBO/CSCV（Probability of Backtest Overfitting）：通过组合交叉验证估计候选池的回测过拟合概率。
- `validation/holdout.py` — holdout 段永久隔离：`--start/--end` 之外的数据不参与搜索与验证循环。

**CLI**

```bash
fz validate overfit <factor> --start 20200101 --end 20241231
```

`fz validate overfit` 直接打印单因子的 IC / IR / Deflated Sharpe p 值 / bootstrap IC 95% 置信区间，不落盘。单因子为 N=1（无多重检验扣减），PBO 仅用于候选池，此命令不计算 PBO。

## Barra 风险模型

**新增能力**

- `risk/style_factors.py` — Barra 风格因子（8 个：`size`、`value`、`momentum`、`volatility`、`liquidity`、`quality`、`growth`、`leverage`）。
- `risk/industry_factors.py` — 行业因子暴露（按 Tushare `stock_basic.industry` 字段分类，one-hot 编码）。
- `risk/exposures.py` + `risk/model.py` — 暴露矩阵组装与风险模型收口（含特质风险 James-Stein 收缩、边际风险贡献分解）。
- `risk/covariance.py` — Newey-West 协方差估计（支持可配置 `--nw-lags`、`--cov-half-life`、半衰期衰减）。

**CLI**

```bash
fz risk build --start 20200101 --end 20241231 [--universe all_a --cov-half-life 90 --nw-lags 2 --spec-shrinkage 0.3 --spec-half-life 90]
```

产物写入 `workspace/risk_models/{run_id}/`。

## 组合优化与归因

**新增能力**

- `portfolio/optimizer.py` + `portfolio/constraints.py` — cvxpy 因子形式 mean-variance QP（CLARABEL solver）：
  - box 约束（`--w-max`）、预算约束（权重和 = 1）、换手约束（`--turnover`）。
  - 行业中性 + 风格中性约束（`--industry-neutral`）。
- `attribution/brinson.py` — Brinson 多期归因（配置/选股/相互作用效果分解）。
- `attribution/risk_attribution.py` — 风险因子归因（持仓风格暴露 × 因子收益）。

**CLI**

```bash
fz portfolio build --start 20200101 --end 20241231 --alpha-file alpha.parquet --industry-neutral [--lam 1 --w-max 0.05 --turnover 0.3]
```

`--alpha-file` 为 `fz mine export-alpha` 产出的 `(ts_code, alpha)` 两列截面 parquet。`portfolio build` 在 `--end` 单日建仓（信号日 = end），解一次 QP，产物为单截面 `weights.parquet`（列 `ts_code/target_weight/prev_weight`）+ `attribution.csv` + `risk_summary.csv` + `manifest.json`，写入 `workspace/portfolios/{run_id}/`。

**MVP 限制**

- 行业中性约束基于等权基准，非真实指数成分权重。
- 收益归因（Brinson）依赖持仓期收益数据；数据不完整时部分分解项为空。

## 单 Agent 挖掘

**新增能力**

- `agents/orchestrator.py` + `agents/nodes.py` — 零外部依赖自建 LLM 闭环（假设 → 生成表达式 → 护栏过滤 → IC 验证 → critic 反思 → 迭代）。
- `agents/memory.py` Negative RAG：把历史失败案例注入 prompt，引导模型跳出重复错误。
- `agents/manifest.py` 因子候选落 manifest（表达式 / IC / 迭代号 / 拒绝原因）可审计。

**CLI**

```bash
fz mine agent --start 20200101 --end 20231231 [--iterations 5]
```

## 多 Agent 团队

**新增能力**

- `agents/roles/` — 4 个角色 Agent 分工：Hypothesis（假设生成）/ Coder（表达式实现）/ Critic（否决审查）/ Librarian（记忆检索）；IC 评估为 `agents/evaluation.py` 提供的评估环节（由编排器调用，非角色类）。
- `agents/team_orchestrator.py` — 跨轮协调：Critic 可在任意轮否决当前候选并要求重新假设。
- `agents/experiment_index.py` — 跨 session 长期记忆（已探索假设 / 成功因子 / 失败模式 JSON 索引）。

**CLI**

```bash
fz mine team --start 20200101 --end 20231231
```

团队产物落 `workspace/mine_team/`。

## 模拟交易 + 成果展示

**新增能力**

- `sim/engine.py` — 多周期权重回测：读取 `workspace/portfolios/{run_id}` 的目标权重，对齐行情数据，扣除换手成本（内部 CostModel），输出净值序列与绩效指标（年化收益、夏普、最大回撤、卡尔玛、换手率、信息比率）。产物 `nav.parquet` + `metrics.json` + `manifest.json` 写入 `workspace/sim/{run_id}/`。
- `reports/portfolio_report.py` — 组合绩效 HTML Dashboard：指标卡 + 净值曲线 + 月度收益热图 + Brinson 归因条形图 + 风险摘要，默认输出 `workspace/reports/portfolio_<run_id>.html`。

**CLI**

```bash
fz sim run --portfolio-dir workspace/portfolios/<id> --start 20200101 --end 20241231
fz sim show --sim-dir workspace/sim/<id>
fz report portfolio --sim-dir workspace/sim/<id> --portfolio-dir workspace/portfolios/<id> [--out workspace/reports/portfolio_<id>.html]
```

成本走内部 CostModel，`fz sim run` 不暴露 `--cost-rate` 参数。

## 端到端主线链路（v0.3.0）

Step 0 为前置数据拉取，Step 1-6 为研究主线：

```bash
# 0. 拉数据（前置）
fz data fetch daily --start 20200101 --end 20241231
fz data fetch daily-basic --start 20200101 --end 20241231

# 1. 挖因子
fz mine search --start 20200101 --end 20231231 --method genetic --trials 200 --top-k 10
# 或: fz mine team --start 20200101 --end 20231231

# 2. 导出 alpha（把某个候选算成单截面 (ts_code, alpha)）
fz mine export-alpha --session workspace/mining_sessions/session_42_genetic --rank 1 --date 20231229 --out alpha.parquet

# 3. 防过拟合验收（可选）
fz validate overfit <factor> --start 20200101 --end 20241231

# 4. 建风险模型
fz risk build --start 20200101 --end 20241231

# 5. 组合优化建仓
fz portfolio build --start 20200101 --end 20241231 --alpha-file alpha.parquet --industry-neutral

# 6. 模拟交易 + 成果展示
fz sim run --portfolio-dir workspace/portfolios/<id> --start 20200101 --end 20241231
fz report portfolio --sim-dir workspace/sim/<id> --portfolio-dir workspace/portfolios/<id>
```

## 明确边界与诚实说明

1. **行业中性约束**：相对等权基准，非真实指数成分权重；接入真实指数成分权重可作为后续增强（当前未实现，无预留接口）。
2. **LLM Agent**（单/多 Agent 挖掘）：依赖外部 LLM API，不内置模型；在无 API key 环境下 Agent 命令不可用，其余功能不受影响。
3. **模拟交易**：基于历史复权价，不考虑盘口深度、部分成交与实盘滑点；不接实盘 OMS，不做实盘下单。
4. **跟踪误差约束**：当前组合优化不内置跟踪误差约束（TEV），需手动通过换手约束近似控制主动风险。
5. **样本内组合合成**：`research/combination/` 仍是实验性工具，样本内 IC 估权重的结果不能称为 OOS 能力。
